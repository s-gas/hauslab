Today I realized that **svcmonitor** had a memory leak. And this was thanks to **sysmetrics** showing a continuos increas of RAM usage. My service was useful, yay!

The leak was caused by the fact that i was checking the wrong condition for closing `resp.Body`.

I also added a Grafana dashboard for **svcmonitor**, so i can see the status of the services from there!

======

Finally looked at how *multi-stage images* work in Docker. Managed to reduce the image size of the Go images from more than 1GB to 200MB using Debian in the 2nd stage. I could potentially reduce the image even more with Alpine, but i was running into problems, because it does not have some dependencies that I need.

I also follow the recommended way of creating Go images, which means to copy `go.mod` and `go.sum` and run `go mod download` before and then copy the rest. In that way any change in code it will not trigger Docker to download again dependencies.
