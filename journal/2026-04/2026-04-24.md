# 24-04-2026

Today I containerized the Go apps as well and kept a docker compose file per service, in this way they are isolated and it's easier to start and stop them individually. But that also meant that I had to define some external Docker networks for enabling communication through them. Port mapping to the host could have been also an option, but that wouldn't have solved the fact that **svcmonitor** sends requests to some ports, and now that that service is containerized, these ports would be internal to its Docker network. Plus, in this way, I don't have anything exposed to host, which is definitely a more secure approach.

So far I'm also segmenting the external Docker network by keeping inside these networks only the services that actually requires to communicate. This means that for now I have two networks: **monitor**, which englobes all the services, and **sysmetrics-observability**, which contains all the services apart from **svcmonitor**. Not sure if this approach would become hard to maintain when I'll have more services, we'll see.
