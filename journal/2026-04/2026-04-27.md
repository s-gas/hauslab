# 27-04-2026

Realized that the preconfigured upstream DNS server set on AdGuard was failing.
Set **Quad9** and **CloudFare** as upstream DNS servers. They both use **DoH**, which means DNS over HTTPS, which means that they send encrypted request over HTTP to upstream.
Learned that what you set as DNS server does not actually do DNS resolution, but the flow is:

device -> DNS server (AdGuard) -> Upstream DNS server -> Root DNS server -> TLD (Top Level Domain) server (for example .com) -> Authorative server

The command for checking DNS lookup is:

```bash
dig [@server] <domain>
```
