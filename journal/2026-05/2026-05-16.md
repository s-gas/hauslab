# 16-05-2026

Started working on the CLI for gasmetrics. For now I'm just gonna go with the simple route, having the CLI as a separate service which sends requests to gasmetrics, and gasmetrics becomes also an HTTP server that listens to these requests. In order to do so I would need a second goroutine for the server.

A very good way to improve this would be to untangle everything and having different services each with their own responsability:
- telegram bot that polls telegram and sends requests (client 1)
- CLI that also sends requests (client 2)
- server that connects to the database and listens for requests, validates them and if valid execs in the database
