Since now I have some simple go tests, I could implement a very simple CI with GitHub actions so that the tests are run at every push. When I will expose the server to the internet I could use GitHub Actions for automating the deployment so that I can replace the cronjob that is running right now.

=====

Today I learned that a useful way for creating a CI/CD workflow is by using Github Action as CI, which means building the images and pushing them to Docker Hub. In order to do so, it is necessary to also login to Docker Hub through Github Actions, this is possible by adding the Docker Hub credentials as secrets to the Github repo. Such a nice feature!

The CD part would be handled by Watchtower, which is a service that basically keeps polling Docker Hub for new images and in case if finds them, it pulls them and redeploys the containers. In this way I could say bye bye to my shell script.
